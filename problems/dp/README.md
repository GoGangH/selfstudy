# Dynamic Programming (다이나믹 프로그래밍)

## 개념
다이나믹 프로그래밍(DP)은 복잡한 문제를 간단한 여러 개의 부분 문제로 나누어 푸는 방법입니다.
중복되는 부분 문제들의 결과를 저장하여 재사용함으로써 효율성을 높입니다.

## 핵심 원리
1. **최적 부분 구조(Optimal Substructure)**: 큰 문제의 최적해가 작은 문제의 최적해로 구성됨
2. **중복되는 부분 문제(Overlapping Subproblems)**: 같은 부분 문제가 여러 번 계산됨

## 접근 방법
- **Top-Down (Memoization)**: 재귀 + 메모이제이션
- **Bottom-Up (Tabulation)**: 반복문 + 테이블

## 대표 문제 유형
- 피보나치 수열
- 최장 증가 부분 수열 (LIS)
- 배낭 문제 (Knapsack)
- 최단 경로 문제
- 편집 거리
- 행렬 연쇄 곱셈
